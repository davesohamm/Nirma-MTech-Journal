| Concept | One-line Description |
|--------|----------------------|
| **OLTP vs OLAP** | OLTP handles transactional workloads; OLAP handles analytical queries. |

| **Page-Oriented Storage Engines** | Data is organized into fixed-size pages for efficient disk access. |

| **Point Query Complexity** | Time taken to find a single specific data item. |
| **Range Query Complexity** | Time to retrieve a continuous range of values. |
| **Efficient Data Retrieval & Amortization** | Techniques to optimize repeated operations over large datasets. |
| **Multidimensional Indexing** | Structures to search across multiple attributes. |
| **Point Queries** | Querying data for a single exact match. |
| **Partial Match Queries** | Querying when only some dimensions are specified. |
| **Range Queries** | Searching data across continuous ranges in one or more dimensions. |
| **Nearest Neighbour Queries** | Finding the closest data points based on distance metrics. |
| **Where-am-I Queries** | Locating the position of a point in high-dimensional space. |
| **KD-Tree** | A spatial partitioning structure used for efficient nearest-neighbour search. |

- Sir taught us about **Apache Spark**.
- Spark is a **big data processing framework** designed for fast in-memory computation.
- It supports iterative and real-time analytics better than traditional Hadoop MapReduce.
- It uses **RDDs (Resilient Distributed Datasets)** to store data in memory across nodes.
- Sir also introduced **SparkContext**, which is the main entry point to Spark execution.
- We briefly discussed **GraphX**, a Spark API used for graph processing and analytics.
- Although this topic isn't part of the exam, it is **highly relevant in industry** and useful to know.

  - **K-Means Algorithm:**
